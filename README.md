# Mope.io Issue Tracker

Public issue tracker for [Mope.io](https://mope.io), owned and operated by
[3AM Experiences LLC](https://3amexperiences.com).

> Mope.io is closed-source. This repository contains no game code. It exists
> solely for bug reports, visual issues, balance feedback, and gameplay
> suggestions from players and testers.

---

## 🚨 The Golden Rule: One Issue = One Problem 🚨

**Do not create massive lists of unrelated bugs in a single issue.**

If you find a bug with the Pelican *and* a bug with the Yeti, open **two
separate issues**. Mega-lists of unrelated problems are impossible for
developers to label, prioritize, track, or close — and they **will be closed**
with a request to resubmit properly.

---

## How to Report

1. **Search first.** Check [open](../../issues) and
   [closed](../../issues?q=is%3Aissue+is%3Aclosed) issues to make sure your
   problem hasn't already been reported. If it has, leave a 👍 or a comment
   with new information on the existing issue.
2. **One issue per report.** If you found 10 bugs, open 10 issues.
3. **Use the template.** When you click **New Issue** you will be prompted to
   choose *Bug Report* or *Feature / Balance Request*. Fill out every required
   field — blank issues are disabled.
4. **Be specific.** Include the animal name, the biome, steps to reproduce,
   and what went wrong. Vague reports like "ability is broken" can't be acted
   on.
5. **Include evidence.** Screenshots, GIFs, or short video clips make reports
   dramatically easier to confirm.

## What Belongs Here

| ✅ Yes | ❌ No |
|---|---|
| Bug reports (one per issue) | Giant mixed lists of unrelated bugs |
| Visual / animation / asset issues | Account, ban, or payment support — use [Support Tickets](https://support.vexxusarts.com/submit-ticket/2-mopeio) |
| AI / pathing problems | Public exploit or cheat instructions — report privately on [Discord](https://discord.com/invite/nQAVB9c) |
| Balance feedback | Vague complaints with no detail |
| Focused gameplay suggestions | Off-topic discussion |

## Labels

Maintainers use labels to categorize and track issues. You don't need to
apply them yourself — they're listed here so you understand what they mean
when you see them on your report.

| Label | Meaning |
|---|---|
| `bug` | Confirmed unintended behavior |
| `visual` | Asset, animation, layering, opacity, or scale issue |
| `gameplay` | Ability, mechanic, or interaction issue |
| `ai` | NPC / bot behavior issue |
| `balance` | Works as coded but feels too strong / weak |
| `suggestion` | New feature or design change request |
| `regression` | Something that used to work but is now broken |
| `needs-triage` | Awaiting maintainer review |
| `needs-info` | More detail needed from the reporter |
| `confirmed` | Reproduced and queued for a fix |
| `duplicate` | Already reported — linked to original |
| `wontfix` | Intentional behavior or out of scope |
| `fixed` | Resolved in a game update |

---

## Issue Lifecycle

```
Opened
  → Triaged (labeled)
      → Confirmed → In Progress → Fixed (closed)
      → Needs Info → (waiting on reporter) → Closed if stale after 14 days
      → Duplicate → Closed with link to original
      → Won't Fix → Closed with explanation
```
